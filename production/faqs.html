Dos and Don'ts

Dos

  • Read the documentation before starting customization.
  • Use the latest version of Bootstrap 4 for compatibility.
  • Keep your dependencies updated.
  • Follow best practices for HTML, CSS, and JavaScript.
  • Test the template in multiple browsers.
  • Use clear and descriptive class names when customizing.
  • Backup your project before making major changes.
  • Minimize unused components to improve performance.
  • Report issues or bugs to the repository maintainer.
  • Respect the license terms for third-party libraries.
  • Optimize images and assets for faster loading.
  • Document your customizations for future reference.
  • Use version control (like Git) for your project.
  • Ensure accessibility for all users.
  • Regularly review and refactor your code.

Don'ts

  • Don't remove license or attribution files.
  • Don't use outdated dependencies.
  • Don't hard-code sensitive information.
  • Don't ignore browser compatibility issues.
  • Don't overload pages with unnecessary plugins.
  • Don't neglect mobile responsiveness.
  • Don't skip code reviews and testing.
  • Don't leave unused code or files in the project.
  • Don't modify core files without backups.
  • Don't overlook performance optimization.
  • Don't use untrusted third-party scripts.
  • Don't duplicate code unnecessarily.
  • Don't expose admin interfaces publicly without security.
  • Don't commit secrets or passwords to your repository.
  • Don't ignore user feedback or bug reports.
